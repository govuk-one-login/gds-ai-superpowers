# A07:2025 Authentication Failures — deep

Pull when an authentication finding is contested or applicability is unclear.

**Check**
- Credential handling: slow salted hashing (argon2/bcrypt), no plaintext, breached-password
  check, no low-entropy limits that block strong passphrases.
- Session management: server-side or signed tokens, secure+httponly cookies, rotation on
  privilege change, idle + absolute timeouts, invalidation on logout.
- MFA available/enforced for sensitive roles; step-up auth on sensitive actions.
- Anti-automation on login (rate limit, lockout/backoff) without enabling user-enumeration.

**Pitfalls**
- Predictable or non-expiring session/reset tokens.
- Verbose login errors ("no such user" vs "wrong password") enabling enumeration.
- JWTs accepted with `alg:none` or without signature verification.

**Example / anti-example**
- *Bad:* session id is a sequential integer in a cookie.
- *Good:* a high-entropy signed session token, rotated on login, with idle+absolute expiry.
