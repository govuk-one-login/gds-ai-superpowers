# A02:2025 Security Misconfiguration — deep

Pull when a misconfiguration finding is contested or applicability is unclear.

**Check**
- Hardened, least-functionality config: unused features/ports/services/accounts disabled.
- No default credentials, no sample apps, no verbose stack traces returned to clients.
- Security headers present (HSTS, CSP, X-Content-Type-Options) for web surfaces.
- Cloud storage/buckets/queues are private by default with explicit, least-privilege grants.
- Config is managed/version-controlled and identical across environments (no prod drift).

**Pitfalls**
- A storage bucket or DB left world-readable "temporarily".
- Debug/verbose error mode enabled in production, leaking internals.
- Inherited insecure framework defaults never reviewed.
- Permissive CORS (`*`) on an authenticated API.

**Example / anti-example**
- *Bad:* an S3-style bucket with public-read ACL holding user uploads.
- *Good:* bucket private, access via signed URLs scoped per-object and time-limited.
